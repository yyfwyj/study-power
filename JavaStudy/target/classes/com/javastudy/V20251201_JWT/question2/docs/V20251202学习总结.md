1. 今天学习到了什么？
首先，是JWT的过滤，具体理论我已经忘记的差不多了
大概的复述一下，就是当Token通过我们的JWTUtils文件生成好，发送给前端，前端保存在本地
然后，前端在每一次请求的时候，都携带到请求头上，用于验证用户身份与合法性
然后，我们需要在网关处或者是在JWTFilter这种文件里去鉴权
2. 如何鉴权？
首先，我们需要创建一个文件：JWTFilter，这个类会继承OncePerRequestFilter，继承这个类的原因是，避免请求多次被验证，同时避免请求信息被多次重新赋值，导致异常
然后，我们要制定一个白名单，过滤我们不需要验证的请求、资源，避免服务器多余消耗
例如：List<String> apiList = Arrays.asList( "/api/login"...);
我们还需要重写一个方法：ShouldNotFilter，这个方法用于过滤请求，如果符合白名单要求，则直接放行
里面我们要使用多个API
例：
proteced boolean shouldNotFilter(HttpSevletRequest request) {
    // 1. 使用[request.getRequestUri]获取uri
    // 2. 使用[request.getMethod]获取请求方法类型，过滤OPTION
    // 如果是OPTION或者是白名单请求，则直接放行，无需鉴权
}
3. 为什么要直接放行OPTION？
OPTION这个请求方法类型，是浏览器在执行CORS预检，但是我不知道CORS预检是什么意思，反正就是判断有没有跨域嘛，肯定不会拦截的
